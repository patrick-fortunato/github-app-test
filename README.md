# github-app-test

Node.js starter app for incrementing semantic versions in a target Node.js repository after push events.

## What this app does

- Reads a push event payload from `GITHUB_EVENT_PATH`
- Optionally filters by source repository (`SOURCE_REPO`)
- Fetches `package.json` from a target repository
- Bumps `version` using `major`, `minor`, or `patch`
- Commits the updated file back to the target repository

This is designed as a lightweight base you can convert into a GitHub App flow (for example by replacing `GITHUB_TOKEN` with an installation token generated by your app).

## Required environment variables

- `GITHUB_EVENT_PATH`: path to GitHub event JSON payload
- `GITHUB_TOKEN`: token with contents write access to target repository
- `TARGET_REPOSITORY`: `owner/repo` of the Node.js repository to update

## Optional environment variables

- `SOURCE_REPO`: only process pushes from this source repo (`owner/repo`)
- `TARGET_BRANCH`: branch in target repo (default: `main`)
- `TARGET_PACKAGE_PATH`: package file path (default: `package.json`)
- `VERSION_BUMP`: `patch` (default), `minor`, or `major`
- `COMMIT_MESSAGE`: custom commit message
- `COMMITTER_NAME` and `COMMITTER_EMAIL`: optional committer identity
- `DRY_RUN=true`: simulate update without committing

## Run locally

```bash
npm install
npm test
npm start
```
